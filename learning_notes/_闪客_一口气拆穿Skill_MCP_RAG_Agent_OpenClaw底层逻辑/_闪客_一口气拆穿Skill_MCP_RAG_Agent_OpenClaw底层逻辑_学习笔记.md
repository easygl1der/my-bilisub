## 🧠 AI 深度分析

好的，这是对视频内容的分析和学习笔记：

## 📋 视频概览
- **核心主题**: 深入解析AI大模型中的关键概念，包括LLM、Prompt、Context、Memory、Agent、RAG、Search，并阐述了它们之间的关系以及如何通过Agent、Skill、Workflow等框架来构建更强大的AI应用。
- **内容类型**: 科普/教程

## 📖 内容概要（150-250字）
本视频旨在科普AI大模型领域的关键术语，并梳理了它们之间的联系，以帮助观众理解AI技术的发展脉络。视频首先介绍了LLM（大语言模型）的概念，引出了Prompt、Context、Memory、Agent等核心概念，并生动地通过一些例子解释了它们的作用。例如，通过模拟对话场景，展示了Prompt的作用，以及Context如何提供信息，Memory如何存储和回忆。视频重点介绍了Agent的概念，将其视为能够调用工具、执行任务的智能实体，并详细解释了RAG（检索增强生成）和Search等技术如何增强Agent的能力。此外，视频还讲解了Langchain、Workflow、Skill等框架和概念，展示了如何通过这些工具来构建更复杂、更强大的AI应用，并强调了未来AI发展将更注重便利性和可控性。

## 🎯 核心要点
- **LLM（大语言模型）**: 视频开篇即引入LLM，并将其发展历程以泡泡图的形式展示，从GPT-1到GPT-4，参数规模不断增大，能力也随之增强。
- **Prompt（提示词）**: 强调了Prompt是与大模型沟通的关键，它包含了用户希望模型执行的任务和背景信息，是指导模型生成输出的重要输入。
- **Context（上下文）**: 指用户与AI的对话历史或相关背景信息，它能够帮助AI更好地理解用户意图，生成更相关的回答。视频中将之前的对话历史作为Context展示。
- **Memory（记忆）**: 描述了Memory的作用，即对历史对话进行总结和压缩，以减少Context的长度，从而提高AI处理长对话的能力。
- **Agent（智能体）**: 被定义为能够调用工具（如搜索、文件读取、脚本执行等）来完成任务的实体。Agent能够接收用户的指令，并根据指令选择合适的工具来执行。
- **RAG（检索增强生成）**: 介绍了RAG技术，它通过检索相关信息并将其加入Prompt，来提高生成内容的准确性和可靠性。
- **Search（搜索）**: 强调了Agent需要具备搜索信息的能力，并区分了与传统数据库搜索的不同，即需要使用向量数据库来查找语义相近的片段。
- **Function Calling**: 解释了Function Calling是Agent与大模型之间进行通信的一种方式，通过定义接口格式，让大模型能够调用Agent提供的工具。
- **MCP（Model Context Protocol）**: 是一种模型上下文协议，用于规范Agent与大模型之间的通信，方便程序的解析。
- **Skill**: 被描述为Agent的技能，可以将一系列函数调用封装成一个可执行的脚本，以模块化的方式管理功能。
- **Workflow**: 指通过编程（如Langchain）的方式，将一系列操作组合起来，形成一个数据处理或任务执行的流程，降低了开发门槛。
- **SubAgent**: 是一种用于处理独立子任务的Agent，可以将复杂任务分解，提高Agent的处理效率和灵活性。
- **Token**: 是衡量大模型计算成本的基本单位，但随着技术发展，Token成本有望降低，使得更强大的模型能够更便捷地部署。
- **Agent的通用性**: 强调了Agent的未来发展趋势是通用性，能够通过配置和调用各种工具来解决多样化的任务。

## 💡 关键概念
| 概念 | 解释 |
|------|------|
| **LLM (Large Language Model)** | 大语言模型，是AI领域的核心，能够理解和生成人类语言，参数量巨大，能力不断提升。 |
| **Prompt (提示词)** | 用户向AI模型输入的指令或问题，是指导模型生成输出的关键。 |
| **Context (上下文)** | AI在进行交互时所处的环境信息，包括对话历史、用户偏好等，用于帮助AI理解意图。 |
| **Memory (记忆)** | AI存储和回忆信息的能力，可以用于长对话的理解和任务的连贯性。 |
| **Agent (智能体)** | 能够自主思考、规划并调用工具（如API、脚本）来完成任务的AI系统。 |
| **RAG (Retrieval-Augmented Generation)** | 检索增强生成，一种结合了信息检索和大模型生成的技术，用于提高生成内容的准确性。 |
| **Search (搜索)** | Agent获取信息的一种方式，通常利用向量数据库进行语义相似性搜索。 |
| **Function Calling** | 一种机制，允许AI模型调用外部函数或工具来执行特定任务，并按照预定义的格式返回结果。 |
| **MCP (Model Context Protocol)** | 模型上下文协议，用于规范Agent与大模型之间的通信，实现工具调用和信息交换。 |
| **Skill** | Agent可调用的具体功能模块，通常以脚本（如Python）的形式实现，描述了其功能和使用方法。 |
| **Workflow** | 通过编程（如Langchain）将多个任务或技能组合起来，形成一个可执行的流程，实现复杂任务的自动化。 |
| **SubAgent** | 辅助Agent处理特定子任务的更小单元，提高了Agent的模块化和任务分解能力。 |
| **Token** | 大模型进行计算的单位，其成本是影响模型应用的关键因素之一。 |
